# TaxMeter iOS (App Store 출시 기준) 제품 설계안

## 1) 제품 한 줄 정의
**TaxMeter는 사용자의 현재 값(소득·지출·자산·가입정보)이 제도 기준선까지 얼마나 남았는지(또는 초과했는지)를 거리로 보여주는 개인 세금/복지 네비게이터 앱**입니다.

---

## 2) 출시 전략: MVP → 확장 → 프리미엄

### MVP (App Store 최초 출시)
아래 15개 기준선을 우선 출시 기능으로 고정합니다.

1. 금융소득 2,000만원 기준
2. 건강보험 피부양자 탈락 기준
3. 연금저축 400만원 세액공제 한도
4. IRP 포함 700만원 통합 한도
5. 기타소득 300만원 기준
6. 간이과세자 매출 8,000만원 기준
7. 해외주식 250만원 공제 한도 기준
8. 신용카드 총급여 25% 기준
9. 청년도약계좌 대상 여부
10. 근로장려금 대상 여부
11. 월세 세액공제 대상 여부
12. 종합소득세 신고 필요 여부
13. ISA 가입 가능 여부
14. 국민연금 최소 수령 조건
15. 청약 가점 기본 계산

### 확장 단계
- 건강보험·연말정산·청년정책의 세부 조건 분기 확대
- 맞벌이 공제 분배 최적화
- 투자(배당/해외/가상자산) 세금 시뮬레이터

### 프리미엄 단계
- 개인 맞춤 “다음 달 행동 추천” 자동 생성
- 다중 시나리오 비교(직장인/프리랜서/사업자)
- 기준선 변경 알림(법령 개정 반영)

---

## 3) 핵심 UX: “거리 중심 화면”

### 홈 대시보드
- **가장 가까운 위험 기준선 TOP 3**
- 각 카드에 `현재값 / 기준값 / 남은 거리`
- 색상 신호
  - 초록: 여유 구간
  - 노랑: 근접 구간
  - 빨강: 초과(리스크 발생)

### 기준선 상세 화면
- 판단 결과(해당/비해당)
- 근거 변수(소득, 지출, 가입형태 등)
- “얼마를 줄이거나 늘리면 유리한지” 액션 가이드
- 계산 기준일 및 데이터 최신화 일자 표시

### 입력 최소화 원칙
- 첫 온보딩은 2~3분 내 완료
- 필수 입력만 받고 나머지는 “없음/모름” 허용
- 공공 데이터 연동 전에도 수동 입력으로 즉시 결과 제공

---

## 4) 입력 구조(최소 입력으로 최대 결과)

## 공통 프로필
- 출생연도
- 거주 형태(월세/전세/자가/기타)
- 근로 형태(근로소득자/프리랜서/사업자/복수소득)
- 부양가족 여부(배우자/자녀 수)

## 소득 입력
- 근로소득(연)
- 금융소득(이자+배당, 연)
- 기타소득(연)
- 사업소득(연 매출/비용 단순 입력)
- 해외주식 손익(연)

## 지출/공제 입력
- 연금저축 납입액
- IRP 납입액
- 신용/체크카드 사용액
- 월세 납부액

## 정책/자격 입력
- 청년정책 연령/소득 조건 관련 필드
- 건강보험 가입 상태(직장/지역/피부양자)
- 청약통장 보유 및 납입 횟수

---

## 5) iOS 앱 구조 (출시 안정성 우선)

- 클라이언트: **SwiftUI + MVVM**
- 데이터 계층: Local(온디바이스) + API(기준선 룰 조회)
- 룰 엔진: 서버에서 버전 관리 (`rule_version`, `effective_date`)
- 원격 설정(Remote Config): 기준선 문구/라벨 긴급 수정
- 분석: 퍼널(온보딩→첫 결과→저장→재방문)

### 권장 모듈
- `Onboarding`
- `BaselineDashboard`
- `BaselineDetail`
- `WhatIfSimulator`
- `PolicyEligibility`
- `Settings & Consent`

## 6) App Store 출시 필수 체크리스트

## 제품/기능
- 최소 10개 이상 기준선이 실제 계산 가능해야 함
- 오프라인 상태에서도 최근 저장값 기준 결과 열람 가능
- 계산 근거 및 안내 문구는 “정보 제공용” 고지 포함

## 개인정보/보안
- 민감 정보 최소 수집 원칙
- 저장 데이터 암호화(키체인/보안 저장소)
- 탈퇴 및 데이터 삭제 기능 제공
- 개인정보처리방침 URL 준비

## 심사 리스크 대응
- 의료/법률/세무 “자문” 표현 금지, “참고 정보”로 명시
- 허위·과장 수익 문구 금지
- 스크린샷에 실제 오해 소지 수치 대신 예시 데이터 사용

## 메타데이터
- 앱 이름: `TaxMeter - 세금 기준선 거리`
- 부제: `내 소득과 제도 기준, 한눈에`
- 키워드: `세금,연말정산,건강보험,금융소득,청년도약계좌,절세`
- 카테고리: Finance

## 8) 릴리즈 로드맵 (8주 예시)

- 1~2주차: 룰 정의, 데이터 모델, UX 와이어
- 3~4주차: 온보딩·대시보드·상세 구현
- 5주차: 시뮬레이션/알림/저장
- 6주차: QA, 예외케이스, 접근성
- 7주차: TestFlight(내부/외부)
- 8주차: App Store 제출 및 심사 대응

## 10) 바로 실행할 개발 우선순위

1. MVP 15개 기준선 룰 테이블 확정

5. App Store 심사용 메타데이터/스크린샷 패키지 제작
